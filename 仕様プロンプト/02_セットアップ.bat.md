# [2/30] セットアップ.bat の仕様

## 指示

- **この仕様だけを読んで `セットアップ.bat` を完全な形で実装し、ファイル全文を出力すること。**
- **`pass`・`TODO`・「省略」・「要約」・「以下同様」などは禁止。** バッチの全行を、実際にそのまま保存して動作する完成形で出力すること。
- **出力が途中で切れた場合は、ユーザーが「続き」と言ったら、続きを最後まで完全に出力すること。**
- これは Windows のバッチファイル（`.bat`、cmd.exe 用）であり、Python ソースではない。Qt/Mixin/facade などの Python 規約は本ファイル自体には直接関係しないが、このバッチが用意する環境上で動くアプリ群の前提として以下を踏まえること。

### アプリ全体の前提（本ファイル関連分）

- アプリ本体は **Python 3.10+ / GUI=PySide6(Qt6)**。GUI は matplotlib の `qt_compat` 経由で Qt を取得する設計（PyQt6 でも同じコードで動作）。
- 依存パッケージは同じフォルダの `requirements.txt` で管理され、`scipy` はオシロ解析（ピーク検出・FFT）に使用、`charset-normalizer` は文字コード自動判定に使用。`spicelib` は LTspice 連携（任意）で、このセットアップでは併せてインストールする。
- このバッチは **専用の仮想環境を `C:\.venv` に作る**。アプリ起動側（`起動.bat`）は `C:\.venv\Scripts\pythonw.exe` を使ってアプリを起動するため、本バッチが作るパスと整合していなければならない。

---

## ファイルの役割／責務

- アプリの**初回セットアップ用バッチ**。`C:\.venv` に Python 仮想環境（venv）を新規作成し、`pip` を最新化したうえで、`requirements.txt` の依存パッケージと追加で `spicelib` をインストールする。
- 実行はバッチをダブルクリックする想定。処理中はユーザーに進行状況をメッセージ表示し、最後に「`起動.bat` をダブルクリックでアプリを起動できる」旨を案内して `pause` で待機する。
- 仮想環境の作成に失敗した場合はエラーメッセージを出して終了コード 1 で抜ける。

---

## 依存（前提となる外部要素）

- **cmd.exe（Windows バッチ）**。POSIX シェルや PowerShell ではない。
- システムにインストール済みの **Python（優先は Python 3.12）**。具体的には以下の探索順:
  1. `%LocalAppData%\Programs\Python\Python312\python.exe`（ユーザー単位インストールの公式 Python 3.12）
  2. 上記が存在しなければ `PATH` 上の `python`
- 同じフォルダ（バッチと同一ディレクトリ）に **`requirements.txt`** が存在すること。
- ネットワーク接続（pip で PyPI からパッケージ取得）。

## 公開API（バッチの全構造・全行の仕様）

バッチには関数定義はなく、上から順に実行される単一フローである。以下の**順序・内容を厳密に**再現すること。

1. `@echo off`
   - コマンドのエコー表示を抑制（先頭行）。

2. `cd /d "%~dp0"`
   - カレントディレクトリを**バッチ自身のあるフォルダ**へ移動。`%~dp0` はバッチのドライブ＋パス。`/d` でドライブも跨いで移動。ダブルクォートで囲み、空白を含むパスにも対応。

3. 区切り線とメッセージ（`echo` 3〜5 行）。**文言を以下の値そのまま**で出力すること:
   ```
   echo ===========================================================
   echo  Creating virtual environment at C:\.venv and installing
   echo  dependencies. This may take a few minutes...
   echo ===========================================================
   ```
   - 1 行目と 4 行目の区切り線は**等号 `=` を 59 個**並べたもの（上の文字列をそのまま使う）。
   - 2 行目・3 行目は先頭に半角スペース 1 個。

4. Python 実行ファイルの解決:
   ```
   set "PY=%LocalAppData%\Programs\Python\Python312\python.exe"
   if not exist "%PY%" set "PY=python"
   ```
   - 環境変数 `PY` に Python 3.12 の標準ユーザーインストールパスを設定。
   - そのファイルが存在しなければ `PY` を `python`（PATH 解決）にフォールバック。

5. 仮想環境の作成:
   ```
   "%PY%" -m venv C:\.venv
   ```
   - `%PY%` を使って `C:\.venv` に venv を作成。

6. 作成失敗のガード:
   ```
   if not exist "C:\.venv\Scripts\python.exe" (
     echo [ERROR] Failed to create the virtual environment.
     pause
     exit /b 1
   )
   ```
   - venv の `python.exe`（`C:\.venv\Scripts\python.exe`）が生成されていなければ失敗とみなす。
   - **エラーメッセージは `[ERROR] Failed to create the virtual environment.` の値そのまま。**
   - `pause` でユーザー確認後、`exit /b 1`（終了コード 1）で抜ける。
   - 丸括弧ブロックの記法（`(` を `if` と同じ行、`)` を独立行）を維持すること。

7. pip の最新化:
   ```
   C:\.venv\Scripts\python.exe -m pip install --upgrade pip
   ```
   - **venv 側の python.exe**（フルパス `C:\.venv\Scripts\python.exe`）を使って pip を最新化。`%PY%` ではなく venv の python を直接フルパスで呼ぶ点に注意。

8. 依存インストール:
   ```
   C:\.venv\Scripts\python.exe -m pip install -r requirements.txt spicelib
   ```
   - `requirements.txt`（`-r`）に加えて **`spicelib`** を同一コマンドで追加インストール。`requirements.txt` 内では `spicelib` はコメントアウトされた任意依存だが、本バッチでは明示的に併せて入れる。

9. 完了案内と待機:
   ```
   echo.
   echo Done. Double-click 起動.bat to launch the app.
   pause
   ```
   - `echo.`（空行）。
   - **完了メッセージは `Done. Double-click 起動.bat to launch the app.` の値そのまま**（日本語ファイル名 `起動.bat` を含む）。
   - `pause` で終了前に待機。

末尾に余分な行・コマンドは置かない（最終行は `pause`）。

## 再現に必須の細部・エッジケース・ガード

- **`@echo off` を必ず先頭行に置く。**
- **`cd /d "%~dp0"`** によりダブルクリック実行時もバッチのフォルダがカレントになる（`requirements.txt` を相対パスで解決できるようにするため必須）。これが無いと別ディレクトリから起動した際に `-r requirements.txt` が失敗する。
- **Python の二段フォールバック**: まず Python 3.12 の固定パス、無ければ `python`。`if not exist "%PY%"` の判定で行う。
- **venv 作成成否は「`C:\.venv\Scripts\python.exe` が存在するか」で判定**（`%ERRORLEVEL%` ではなくファイル存在チェック）。失敗時のみ `[ERROR]` を出し `pause`→`exit /b 1`。
- **pip 系コマンドは `%PY%` ではなく venv 内の `C:\.venv\Scripts\python.exe` をフルパスで使う**（作成した環境にインストールするため）。ここを `%PY%` にしてはならない。
- `pip install` を **1 行で `-r requirements.txt spicelib`** とまとめる（2 回に分けない）。
- 既存の `C:\.venv` がある状態で再実行されても `python -m venv` は既存ディレクトリに対して安全に動作する想定（特別な削除処理は入れない）。
- 文字コード: バッチ内の日本語は完了メッセージの `起動.bat` のみ。バッチファイルは **Shift_JIS(CP932)** で保存するのが安全（日本語ファイル名の文字化け・実行不能を避ける）。`chcp` の明示変更は行わない。
- 各 `echo` 行・`if` ブロックの**順序を厳守**（メッセージ → Python 解決 → venv 作成 → 失敗ガード → pip 更新 → 依存導入 → 完了案内 → pause）。
- 文末の句点や `...`（ピリオド 3 つ）、`[ERROR]` の角括弧など、記号も**原文どおり**にすること。

## このファイルに関係する落とし穴

- **これはバッチであり Python ではない**ため、Qt6 スコープ付き列挙・Mixin 規約・facade・monospace 回避・grid linewidth=None 回避といった Python 側の落とし穴は本ファイルの実装には登場しない。ただし下流のアプリが PySide6(Qt6) 前提で動くために、本バッチが `requirements.txt`（`PySide6` を含む）を正しくインストールすることが前提条件となる。
- `起動.bat`（同梱の起動スクリプト）は **`C:\.venv\Scripts\pythonw.exe`** を使ってアプリを起動する。本バッチが作る venv のパス（`C:\.venv`）を変更すると起動側と不整合になるため、**`C:\.venv` というパスは両者で一致させる**こと。
- パスのダブルクォート: 空白を含む可能性のある `%PY%` や `%~dp0` は必ず `"..."` で囲む。`C:\.venv\...` 系は空白を含まないが、統一性のため存在チェックではクォートしてよい。
- `exit /b 1` は**バッチを呼び出し元に戻して終了**する形（`/b`）。ダブルクリック起動時はウィンドウが閉じるが、直前に `pause` を置いているのでユーザーがエラーを読める。
