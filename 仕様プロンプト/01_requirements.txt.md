- `pass`・`TODO`・省略・要約・「以下同様」などは**一切禁止**。コメント行も含めて、ここに書かれたとおり一字一句を再現すること。

### アプリ全体の前提（このファイルに関係する分）

- 本アプリは **Python 3.10+ / GUI=PySide6(Qt6)** のグラフ描画・オシロ解析アプリ。`requirements.txt` はその実行に必要な Python パッケージを宣言する依存定義ファイル。
- GUI は **PySide6 を想定**（公式 Qt for Python・LGPL）。Qt は必ず matplotlib の `qt_compat` 経由で取得する設計のため、**PyQt6 でも同じコードで動作**する（Qt6 系）。よって `requirements.txt` には PySide6 のみを記載し、PyQt6 は記載しない。
- **scipy は遅延 import＋numpy フォールバック**（無くても起動する）設計。ただし依存定義上は scipy を必須側に列挙する（オシロ解析のピーク検出・FFT で使うため）。

## ファイルの役割・責務

`requirements.txt` は **pip がインストールすべき Python パッケージとそのバージョン下限を宣言する依存定義ファイル**。`pip install -r requirements.txt` で本アプリの動作環境を構築するために使う。

- 必須パッケージ（バージョン下限つき）を上部に列挙。
- 各行の末尾またはブロックに**日本語コメント**で、そのパッケージの用途・任意/必須の別・補足を明記する。
- 「任意」セクションを設け、`spicelib`（LTspice 連携用）をコメントアウト状態で示し、有効化条件を説明する。
- 末尾に GUI バックエンドに関する補足（PySide6 想定・PyQt6 でも動作）を記載する。

このファイルは **Python コードではなく、テキストベースの依存リスト**であり、書式は pip の requirements 形式（`パッケージ名[比較演算子バージョン]` ＋ `#` 始まりのコメント）。

### 桁揃え・空白に関する厳密な指定

- `scipy>=1.9` とコメント `#` の間は、`scipy>=1.9` の直後に半角スペースを並べて `#` がちょうど列を揃える位置（`charset-normalizer  #` の `#` と同じ列）に来るようにする。実ソースでは `scipy>=1.9` の後に半角スペース 10 個、`charset-normalizer` の後に半角スペース 2 個で、`#` の開始列が一致している。
- 区切りコメント `# --- 任意 ` の後ろは、行幅を埋めるためのハイフン（`-`）が **71 個**続く（`# --- 任意 ` ＋ `-` ×71 で右端を揃える）。
- `# spicelib` の行は、`spicelib` の後に半角スペース 8 個を空けてから 2 個目の `#` を置き、`# LTspice 連携で波形データを生成する場合のみ必要` と続ける。
- 2 行目・3 行目の継続コメント（`#                 #` で始まる行）は、先頭 `#` の後に半角スペース 17 個、続いて 2 個目の `#` を置き、`spicelib` 行の説明文の列に合わせる。
  - 2 行目: `# （tools/generate_ltspice.py）。LTspice 本体も別途必要。`
  - 3 行目: `# セットアップ.bat では spicelib も併せて入れています。`
- `spicelib` ブロックと末尾の GUI 補足ブロックの間には空行を 1 行入れる。

## 再現に必須の細部・エッジケース・落とし穴

- **PyQt6 は記載しない**: GUI は PySide6 のみを必須として列挙する。matplotlib の `qt_compat` 経由で Qt を取得する設計のため、ユーザーが手動で PyQt6 を入れても同一コードで動くが、依存定義としては PySide6 を正とする。
- **scipy はコメント上「使用」と明記するが、実装側は遅延 import＋numpy フォールバック**で scipy 不在でも起動する。よってバージョン指定は下限 `>=1.9` のみで上限は付けない。
- **charset-normalizer はバージョン指定なし・任意（推奨）**: 文字コード自動判定の精度向上目的。固定バージョンを付けない。
- **spicelib はコメントアウトしたまま**にする（先頭 `#`）。これは LTspice 連携（`tools/generate_ltspice.py`）で波形データを生成する場合のみ必要であり、LTspice 本体も別途必要。`セットアップ.bat` では spicelib も併せてインストールする旨を補足する。
- **バージョン演算子は `>=` のみ**を使用し、`==`（厳密固定）や `~=` は使わない。各パッケージは下限のみ指定。
- **日本語コメントの全角記号**（`（）`・`・`・`。`）はそのまま全角で記述する。`.bat` 拡張子は半角。`tools/generate_ltspice.py` のパス区切りは半角スラッシュ。
- **区切り線のハイフン本数を変えない**こと（行幅の見た目が崩れるため実ソースの本数 71 を維持）。
- このファイルは pip 形式のため、Python の構文規則（インデント・Qt6 列挙のスコープ・Mixin 規約・facade・monospace 回避・grid linewidth=None 回避 等）は**適用されない**。それらはアプリ本体ファイルの落とし穴であり、`requirements.txt` には無関係。ここで誤って Python コードや追加パッケージを足さないこと。
