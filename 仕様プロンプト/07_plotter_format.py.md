## ファイルの役割 / 責務

SI 接頭辞（工学接頭辞）つきの数値整形・解析ユーティリティを提供する小さなモジュール。

モジュール docstring（1 行）は次の文字列とすること（趣旨を厳密に再現）:

```
"""SI接頭辞つき数値フォーマット（format_eng / parse_eng / eng_125_sequence）。"""
```

提供する公開機能は次の 3 系統:

1. **`format_eng`（= `_eng`）** … float を工学接頭辞付きの短い文字列へ整形する。
2. **`parse_eng`** … `'1ms'` `'500us'` `'2.5'` `'1e-3'` のような文字列を float に変換する。
3. **`eng_125_sequence`** … 下限〜上限を 1-2-5 刻みで並べた表示文字列リストを生成する（オシロのレンジプリセット用）。

ファイル先頭にエンコーディング宣言コメント `# -*- coding: utf-8 -*-` を付けること。

## 公開 API（完全シグネチャと挙動）

### `def _eng(x):`

工学接頭辞付きの簡易表記を返す。docstring は `"""工学接頭辞付きの簡易表記。"""`。

アルゴリズム（厳密に再現すること）:

1. `x == 0` のときは即 `"0"`（文字列）を返す。
2. 次の `(factor, suffix)` のタプルリスト `units` を**この順序のまま**定義する:
   ```
   units = [(1e-12, "p"), (1e-9, "n"), (1e-6, "µ"), (1e-3, "m"),
            (1, ""),     (1e3, "k"),  (1e6, "M"),   (1e9, "G")]
   ```
   - 接頭辞文字は順に: `p`, `n`, `µ`（U+00B5 マイクロ記号。`u` ではない）, `m`, （空文字）, `k`, `M`, `G`。
3. `units` を先頭から走査し、`abs(x) < factor * 1000` を満たす最初の要素で
   `f"{x / factor:.3g}{suf}"` を返す（有効数字 3 桁の `g` 書式 + 接頭辞）。
4. どの要素にも当てはまらない（= `abs(x)` が 1e9*1000 以上）場合は、フォールバックで `f"{x:.3g}"` を返す（接頭辞なし）。

戻り値: `str`。

例: `_eng(0) == "0"`、`_eng(1e-3)` は `"1m"`、`_eng(1500)` は `"1.5k"`、`_eng(2e6)` は `"2M"`。

### `format_eng = _eng  # 公開名`

`_eng` を `format_eng` という公開名に**別名束縛**する（同一オブジェクトを指す代入文。`def` の再定義ではない）。コメント `# 公開名` を残す。後述の `eng_125_sequence` 内ではこの公開名 `format_eng` を呼ぶ。

### モジュール定数 `_ENG_MULT`（辞書）

`parse_eng` が使う接頭辞→倍率の辞書。**キー・値・並びを正確に**:

```
_ENG_MULT = {"p": 1e-12, "n": 1e-9, "u": 1e-6, "µ": 1e-6, "m": 1e-3,
             "k": 1e3, "M": 1e6, "G": 1e9}
```

ポイント:
- micro は `"u"`（ASCII）と `"µ"`（U+00B5）の**両方**を持ち、どちらも `1e-6`。
- `"m"` は milli (`1e-3`)、`"M"` は mega (`1e6`)（**大文字小文字を区別**）。
- `format_eng` 側の `units` と違い、ここには空文字キーや `"G"` 以外の大文字は無い。`"k"` は小文字のみ。

### `def parse_eng(text, default=None):`

`'1ms'` `'500us'` `'2.5'` `'1e-3'` のような入力を float に変換する。docstring は
`"""'1ms' '500us' '2.5' '1e-3' のような入力を float へ変換する。"""`。

アルゴリズム（順序・エッジケースを厳密に再現）:

1. `s = (text or "").strip()` … `text` が `None` でも空文字でも安全に空文字へ正規化してから前後空白除去。
2. `if not s:` のとき `return default`（空入力は既定値）。
3. **まず素の float 化を試す**: `try: return float(s)` … `'1e-3'` `'2.5'` `'-4'` などはここで確定して返す。`except ValueError: pass` で次へ。
4. `import re`（関数内ローカル import）。
5. 正規表現 `r"^\s*([+-]?[\d.]+)\s*([a-zA-Zµ]*)"` で `s` を `re.match`。
   - group(1): 符号付き数値部（数字と `.` の並び）。
   - group(2): 単位部（ASCII 英字または `µ` の 0 文字以上）。
   - マッチしなければ `return default`。
6. `try: num = float(m.group(1)) except ValueError: return default`（数値部が float 化できなければ既定値）。
7. 単位文字列 `m.group(2)` を**先頭から 1 文字ずつ**走査し、最初に `_ENG_MULT` に含まれる文字が見つかったら `return num * _ENG_MULT[ch]`（その接頭辞倍率を掛けて返す）。コメント `# 単位中の SI 接頭辞を探す`。
8. 接頭辞が 1 つも見つからなければ `return num`（倍率なし）。コメント `# 単位のみ（例 '2V'）は倍率なし`。

戻り値: `float`、または解析不能時は `default`（既定 `None`）。

挙動の要点 / エッジケース:
- `'1ms'` → 単位 `"ms"` の先頭 `"m"` が `1e-3` にヒット → `1 * 1e-3 = 0.001`。
- `'500us'` → `"us"` の先頭 `"u"` が `1e-6` → `500e-6 = 0.0005`。
- `'2V'` → `"V"` は `_ENG_MULT` に無い → 倍率なしで `2.0`。
- `'1e-3'` → ステップ 3 の `float('1e-3')` で即 `0.001`（正規表現には進まない）。
- `'2MHz'` → `"MHz"` の先頭 `"M"`(=mega) がヒット → `2e6`（`"H"` `"z"` には到達しない）。

### `def eng_125_sequence(lo, hi, suffix=""):`

`lo`〜`hi` を 1-2-5 刻みで並べた表示文字列リストを返す（オシロのプリセット用）。docstring は
`"""lo〜hi を 1-2-5 刻みで並べた表示文字列のリストを返す（オシロのプリセット用）。"""`。

アルゴリズム:

1. `seq, dec = [], -12` … 空リストと開始指数（`10^-12` から）。
2. `while 10.0 ** dec <= hi * 1.0001:`（上限側に微小マージン 1.0001 を掛けて判定）ループ:
   - `for m in (1, 2, 5):`（仮数 1, 2, 5 をこの順）:
     - `v = m * 10.0 ** dec`
     - `if lo * 0.9999 <= v <= hi * 1.0001:`（下限側は 0.9999、上限側は 1.0001 のマージン）なら
       `seq.append(format_eng(v) + suffix)` … 整形文字列に `suffix` を連結して追加。
   - `dec += 1`
3. `return seq`

戻り値: `list[str]`。要素は昇順（指数小→大、各指数内で 1→2→5）。

要点:
- マージン定数は**下限 0.9999 / 上限 1.0001** を正確に使う（浮動小数の境界取りこぼし防止）。
- 整形は `format_eng`（= `_eng`）経由。`suffix` は既定 `""`。例 `eng_125_sequence(1e-3, 1, "s")` は `"1ms" "2ms" "5ms" "10ms" ... "1s"` のような並び（各値に `"s"` が付く）。
- 開始指数は `-12`（pico 相当）固定。

## 再現に必須の細部・落とし穴

- **マイクロ記号の使い分け**: `format_eng` の `units` 内出力は `"µ"`（U+00B5）。`_ENG_MULT` の parse 側キーは `"u"` と `"µ"` の両方。文字コードを取り違えない。ファイルは UTF-8 保存必須。
- **大文字小文字の区別**: `_ENG_MULT` では `"m"`=milli と `"M"`=mega、`"k"`(小文字)=kilo、`"G"`=giga。`format_eng` の `units` にも対応する大小がある。
- **`format_eng` は別名束縛**であって独立 `def` ではない。`_eng` を実装し、その後に `format_eng = _eng` を置く。
- **`parse_eng` の評価順**: ①空判定 → ②`float()` 直接 → ③正規表現 → ④接頭辞走査、の順序を守る。`'1e-3'` が ② で確定する点が重要（`"e"` が単位走査に回ると誤動作する）。
- **`re` は関数内 import**。モジュール先頭で import しない（実コードの構成を保持）。
- **`np` は冒頭 import するが本体未使用**でも残す。
- `format_eng` の最終フォールバック（`abs(x)` が 1e12 以上の巨大値）で接頭辞なし `f"{x:.3g}"` を返す分岐を省略しない。
- 本ファイルは Qt も matplotlib も grid も扱わないため、Qt6 列挙・Mixin・facade・monospace・grid linewidth=None などの注意点は適用されない（依存させないこと自体が要件）。
