# [3/30] 起動.bat の仕様

## 指示

- この仕様だけを読んで、`起動.bat` を**完全な形**で実装し、その全文を出力すること。
- `pass`・`TODO`・省略・要約・「ここは元コード参照」等は**一切禁止**。すべての行を実際の内容で書くこと。
- 出力が途中で切れた場合は、ユーザーが「続き」と入力したら、続きを最後まで出力すること。
- これは Windows のバッチファイル（`.bat`、`cmd.exe` 用バッチスクリプト）であり、文字コードは ASCII 範囲のみ（日本語文字列は含まない）。改行は CRLF を想定。

### アプリ全体の前提（本ファイルに関係する範囲）

- 本アプリは Python 3.10+ / GUI=PySide6(Qt6) で動くデスクトップグラフアプリで、エントリポイントは同一ディレクトリにある `graph_app.py` である。
- 起動には専用の仮想環境 `C:\.venv` を使う想定で、その中の `pythonw.exe`（コンソールウィンドウを出さない Python ランチャ）で `graph_app.py` を起動する。
- 仮想環境が存在しない環境でも、システム既定の `pythonw` にフォールバックして起動できるようにする（環境非依存で動くこと）。

---

## ファイルの役割 / 責務

`起動.bat` は、このグラフアプリ（`graph_app.py`）を**ダブルクリック一発で起動するためのランチャ用バッチファイル**である。役割は次の3点に集約される。

1. カレントディレクトリを、このバッチファイル自身が置かれているフォルダへ移動する（`graph_app.py` を相対パスで確実に見つけるため）。
2. 起動に使う Python 実行ファイル（コンソール非表示の `pythonw.exe`）を決定する。専用仮想環境 `C:\.venv\Scripts\pythonw.exe` を最優先で使い、無ければシステムの `pythonw` にフォールバックする。
3. 決定した Python 実行ファイルで `graph_app.py` を起動する。コンソールウィンドウをそのまま残さず、独立したプロセスとして起動する。

GUI アプリを `pythonw.exe`（`python.exe` ではない）で起動するため、黒いコンソールウィンドウは表示されない。

---

## 依存（前提とするもの）

- Windows のコマンドプロセッサ `cmd.exe`（バッチ実行環境）。
- 起動対象 `graph_app.py` が、このバッチファイルと**同じフォルダ**に存在すること。
- 任意: 専用仮想環境 `C:\.venv`（その配下に `Scripts\pythonw.exe`）。存在しなくても動作する。
- フォールバック先として、`PATH` 上に `pythonw`（GUI 用 Python ランチャ）が通っていること。

外部に依存するインストール手順や追加ライブラリの記述は本ファイルには含めない（依存解決は `graph_app.py` 側・仮想環境側の責務）。

## 公開API（バッチには関数定義は無いため、行単位の挙動仕様）

このバッチには `:label` によるサブルーチンや関数定義は存在しない。先頭から末尾まで順に実行される逐次スクリプトである。各行の役割・挙動を精密に示す。

### 1. エコーオフ

```
@echo off
```

- 各コマンド行のエコー表示を抑制する。先頭の `@` により、この `echo off` 行自体もエコーしない。
- これがファイルの**1行目**であること。

### 2. バッチ自身のフォルダへ移動

```
cd /d "%~dp0"
```

- `%~dp0` は「このバッチファイルのドライブ＋パス（末尾に `\` を含む、バッチが置かれたディレクトリ）」に展開される特殊変数。
- `cd /d` の `/d` オプションにより、ドライブが異なる場合でもドライブ文字ごと変更してそのフォルダへ移動する。
- パスにスペースが含まれても壊れないよう、`"%~dp0"` のように**ダブルクォートで囲む**こと（必須）。
- 目的: 以降で `graph_app.py` を相対パス（ファイル名のみ）で参照しても、確実にこのバッチと同じフォルダのものを起動できるようにする。

### 3. コメント行（rem）

```
rem Launch the app with the dedicated virtual environment at C:\.venv
```

- `rem` 行はコメント。専用仮想環境 `C:\.venv` を使ってアプリを起動する旨を英語1行で記す。
- コメント文言は上記の通り**正確に**記載すること（仕様データ）。

### 4. 既定の Python 実行ファイルを変数に設定

```
set "PYW=C:\.venv\Scripts\pythonw.exe"
```

- 環境変数 `PYW` に、専用仮想環境内の `pythonw.exe` への絶対パス `C:\.venv\Scripts\pythonw.exe` を設定する。
- `set "PYW=..."` のように**変数名＝値の全体をダブルクォートで囲む**書式を用いる（値中の末尾空白混入や特殊文字の問題を避けるための慣用形）。
- パス値は**正確に** `C:\.venv\Scripts\pythonw.exe`（先頭ドット付きの隠しフォルダ名 `.venv`、`Scripts` サブフォルダ、`pythonw.exe`）であること。

### 5. フォールバック判定（仮想環境が無ければ素の pythonw）

```
if not exist "%PYW%" set "PYW=pythonw"
```

- `if not exist "%PYW%"`: 直前で設定した `C:\.venv\Scripts\pythonw.exe` が**存在しない**場合に、後続の `set` を実行する。
- フォールバック値は `pythonw`（拡張子なしのコマンド名）。`PATH` 上の GUI 版 Python ランチャを使う。
- ここでも `"%PYW%"` を**ダブルクォートで囲む**こと（パスにスペースが含まれても判定が壊れないように）。
- 分岐は1行で完結（`if ... set ...`）。`else` は使わない。

### 6. アプリ起動

```
start "" "%PYW%" graph_app.py
```

- `start` コマンドで、決定済みの `%PYW%`（`pythonw.exe` か `pythonw`）に `graph_app.py` を引数として渡して**独立プロセスとして起動**する。
- `start` の**第1引数 `""`（空のタイトル）は必須**。`start` は最初のクォート付きトークンをウィンドウタイトルとして解釈するため、これを省くと実行ファイルパスがタイトルと誤認され、アプリが起動しない。空文字タイトル `""` を必ず置くこと。
- `"%PYW%"` はダブルクォートで囲む（パスのスペース対策、必須）。
- `graph_app.py` は相対パス（ファイル名のみ）。2行目で同フォルダへ移動済みのため、これでこのバッチと同じ場所の `graph_app.py` が起動される。
- これが**最終行**。`pause` 等で待機しない（GUI アプリなのでコンソールを残さない）。

## 再現に必須の細部・エッジケース・落とし穴

- **行の順序が重要**: 「変数設定（行4）→ 存在しなければフォールバック（行5）→ 起動（行6）」の順を必ず守る。フォールバック判定は既定値設定の後でなければならない。
- **`cd /d` の `/d` は必須**: バッチがネットワークドライブや C 以外のドライブ上にある場合でも、ドライブごと移動するために必要。
- **`%~dp0` は末尾に `\` を含む**: そのため `cd /d "%~dp0"` は正しくフォルダへ移動する（末尾スラッシュがあっても問題ない）。
- **`start "" ...` の空タイトルは省略不可**: 省略すると `"%PYW%"` がタイトル文字列として消費され、Python が起動しないバグになる。これが最大の落とし穴。
- **`pythonw.exe`（`python.exe` ではない）を使う**: コンソールウィンドウを出さずに GUI を起動するため。`python.exe` にすると黒いコンソールが残る。
- **クォートの徹底**: `"%~dp0"`、`"%PYW%"` はすべてダブルクォートで囲む。パスにスペースや日本語（例: `デスクトップ`）が含まれても壊れないようにするため。本バッチは実際に日本語パス（`...\OneDrive\デスクトップ\グラフ\`）配下に置かれる前提なので、クォートの欠落は致命的。
- **`pause` を入れない**: GUI アプリ起動後すぐにバッチを終了させ、コンソールを残さない設計。
- **`if not exist` の判定対象は `pythonw.exe` ファイル本体**: 仮想環境フォルダの有無ではなく、`pythonw.exe` 実体の有無で分岐する。
- **本ファイルは Python ではない**: Qt/matplotlib 等の Python 側ルール（Qt6 スコープ付き列挙、Mixin 規約、facade、monospace 回避、grid linewidth=None 回避、scipy 遅延 import 等）は本バッチには一切現れない。これらの規約は起動対象 `graph_app.py` 以降のモジュールの責務であり、`起動.bat` はそれらを起動するだけの薄いランチャに徹する。
