# 検討：ライブラリ導入でコード短縮・高精度化できるか

## 結論（先に）
- **コード短縮の余地は小さい**：自前の数値処理の大半は「オシロ測定の専用ロジック（立上り/エッジ/アイ/decimate 等）」で、scipy に直接の代替が無い。積分・窓関数・ピーク検出・STFT・IIRは**既に scipy を使用済み**。
- **高精度化の余地は実測で確認できた**：下記3点は scipy 導入で誤差が明確に小さくなる（しかも新機能も増える）。長さはほぼ同じで精度が上がる。

## 高精度化（実測：`scratchpad/lib_accuracy.py` の生出力）

### ① 指数近似（近似曲線）：log線形化 → `scipy.optimize.curve_fit`
現状の指数フィットは `polyfit(x, log(y), 1)`＝**対数空間の線形回帰**で、ノイズで偏る（小さいyを過大評価）。非線形最小二乗にすると真値に近づく。
```
真値 A=2, B=-3
現状(log線形): A=1.871 B=-2.894 | パラメータ誤差=0.235 | 残差SSE=0.247
curve_fit    : A=1.995 B=-2.983 | パラメータ誤差=0.022 | 残差SSE=0.073
→ パラメータ誤差 10.8倍小さい / 残差SSE 3.36倍小さい
```

### ② 数学チャンネルの補間：`np.interp`(線形) → `scipy.interpolate.CubicSpline`
B系列をA時間軸へ補間する所が線形のみ。3次補間にすると滑らかな信号で誤差が激減。**「補間種別(cubic/sinx-x)」という不足機能も同時に実装できる**。
```
正弦を25点→400点に補間, 真値とのRMSE
線形(np.interp)  : 0.01757
3次(CubicSpline) : 0.00025
→ 3次補間は誤差 69.5倍小さい
```

### ③ フィルタ：1次RC → `scipy.signal.butter` + `filtfilt`（零位相）
現状は1次RC（位相遅れあり・減衰が緩い）。4次Butterworth＋零位相フィルタにすると本体の復元が正確。**digital LPF/HPF/BPF という不足機能も同時に実装できる**。
```
5Hz本体+500Hzノイズをカットオフ50Hzで除去, 真の5Hzとの RMSE
1次RC(現状)            : 0.07455
4次Butterworth+filtfilt: 0.04022
→ 零位相Butterworthは誤差 1.9倍小さい
```

## ついでに“新機能”が増えるライブラリ
- `scipy.signal.correlate`（FFT法）→ **自己相関/相互相関**（不足機能）。`phase_delay` の `np.correlate` も高速化。
- `scipy.signal.welch`（平均ペリオドグラム）→ ノイズ推定が頑健になり **SNR/チャネル電力** の精度・安定性↑。
- `scipy.signal.hilbert` → **包絡線（Envelope）**（不足機能）。
- `scipy.interpolate` → **Deskew/リサンプル** にも流用可。

## 既に最適（変更不要）
- 積分=`cumulative_trapezoid`、微分=`np.gradient`、窓=`scipy.signal.windows`、ピーク=`find_peaks`、平滑=`savgol_filter`、STFT=`scipy.signal.spectrogram`、IIR=`lfilter`。
- `decimate_minmax`（波形表示用の min/max 間引き）は**ライブラリ代替なし**（`scipy.signal.decimate` はスパイクを消すので不可）。自前のままが正解。

## もう一つの解釈：「自前ライブラリを作る」
`analysis_*`/`advanced`/`mathchan`/`plotter_*` は既に**GUI非依存**なので、`pip install` できる**再利用パッケージ（例: `wavelab`）**に切り出せる。これは**移植・再現・テスト**に有利（精度とは別軸のメリット）。

## 推奨
精度・機能の効果が大きい順に、**scipyを遅延import＋無い時はフォールバック**（既存パターン）で導入：
1. **指数近似 → curve_fit**（誤差10.8倍改善・コード長ほぼ同じ）
2. **補間 → CubicSpline**（誤差69.5倍・補間種別UIも追加）
3. **Butterworth + filtfilt**（誤差1.9倍・LPF/HPF/BPF を追加）
4. （任意）correlate/welch/hilbert で 相関・頑健SNR・包絡線 を追加

※ いずれも「コードが大きく短くなる」より「**ほぼ同じ長さで高精度＆機能増**」が正確な評価。
