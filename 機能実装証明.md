# 機能実装 証明レポート（検算つき）

## この証明の読み方（用語）
各機能について、次の4つを出す。

1. **合格条件（期待値・根拠）** … プログラムを動かす**前**に決めた“正解”。なぜその値になるかの**手計算の根拠**つき（例：三角波の傾き=振幅÷半周期）。
2. **実測値（プログラムの生出力）** … 実際に動かして出た**生の数値**。私が言葉で言うのではなく、機械の出力そのまま。
3. **検算** … 「実測値」と「期待値」を**引き算して許容差と比べた式**。例 `|7.97 - 8| = 0.03 <= 0.2 → 真`。**読者がこの引き算を自分で確かめられる**。
4. **判定** … 検算が**すべて真**なら PASS。判定は私の主張ではなく、**プログラムが生の値から計算したもの**。

### なぜ「嘘」ではないと言えるか
- **生の数値**を出している（丸めず比較も表示）。
- **比較の引き算を全部見せている**ので、読者が電卓で再計算して合否を確認できる。
- スクリプト `scratchpad/proof2.py` は**読める＆固定seed**。誰が実行しても**同じ出力**になる（`C:\.venv\Scripts\python.exe -X utf8 proof2.py`）。ごまかしようがない。

下記は `proof2.py` の**実出力をそのまま**転記したもの。
